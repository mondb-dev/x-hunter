# Sebastian D. Hunter

An autonomous AI agent that browses X (Twitter) indefinitely, forms a worldview from scratch via a dynamic belief ontology, and publishes hourly journals + periodic checkpoints to the web.

---

## How it works

The agent runs continuously in 30-minute cycles. Each cycle it:

1. Browses X through an isolated Chrome profile (OpenClaw)
2. Writes an **hourly HTML journal** with observations, screenshots, and footnoted sources
3. Updates internal belief axes (score + confidence) based on what it reads
4. Every 6th cycle: composes and posts a **tweet** on X
5. Once per day: writes a **daily belief report**, may write an **article** or **ponder**, and commits everything to GitHub → Vercel auto-deploys the website
6. Every 3 days: generates a **checkpoint** — a full worldview snapshot

The agent starts with zero ideology and discovers belief axes only when recurring tensions appear across multiple accounts and topics.

## 1. Clone and configure

```bash
git clone https://github.com/mondb-dev/x-hunter.git
cd x-hunter
cp .env.example .env
```

Open `.env` and fill in every value:

```
# LLM — Google Gemini (used by Vertex for articles + generation)
GOOGLE_API_KEY=

# LLM — OpenAI-compatible (checkpoints, ponders; works with OpenAI, xAI/Grok, etc.)
# For Grok: set OPENAI_API_BASE_HOST=api.x.ai and use your Grok key
OPENAI_API_KEY=
OPENAI_API_BASE_HOST=api.openai.com
OPENAI_MODEL=gpt-4o

# X (Twitter) credentials
X_USERNAME=          # X account username (no @)
X_PASSWORD=          # X account password

OPENCLAW_PROFILE=x-hunter

# GitHub — for agent auto-commit + push
GITHUB_TOKEN=        # Personal access token with repo write scope
GITHUB_REPO=mondb-dev/x-hunter
GIT_USER_NAME=x-hunter-agent
GIT_USER_EMAIL=agent@x-hunter.local
VERCEL_DEPLOY_HOOK=  # Optional: Vercel → Project Settings → Git → Deploy Hooks

# Base wallet address for x402 micropayments (USDC on Base mainnet, optional)
PAYMENT_ADDRESS=

# Solana wallet — on-chain identity (generated by setup.sh)
SOLANA_PUBLIC_KEY=
SOLANA_PRIVATE_KEY=
```

> **GitHub token:** go to github.com → Settings → Developer settings → Personal access tokens → Generate new token → select `repo` scope.

Then install dependencies:

```bash
npm install --prefix runner
npm install --prefix scraper
npm install --prefix web
```

## 2. One-time setup

Run this once before the first session:

```bash
bash runner/setup.sh
```

This will:
- Install OpenClaw globally (`npm install -g openclaw@latest`)
- Point OpenClaw's workspace to this project root
- Register the `x-hunter` agent
- Install and start the OpenClaw gateway daemon
- Open the `x-hunter` Chrome profile and navigate to `x.com/login`

**At this point a browser window opens. Log in to X manually, then press ENTER in the terminal to continue.**

The session cookie is saved to the profile. You will not need to log in again unless the session expires.

---

## 3. Deploy the website (one time)

1. Push this repo to GitHub (already done if you cloned from `mondb-dev/x-hunter`)
2. Go to [vercel.com/new](https://vercel.com/new)
3. Import `mondb-dev/x-hunter`
4. Set **Root Directory** to `web/`
5. Deploy — Vercel will auto-deploy on every future push to `main`

---

## 4. Run the agent

```bash
bash runner/run.sh
```

This starts an infinite loop of 30-minute cycles. The script:
- Checks the OpenClaw gateway is running (starts it if not)
- Every cycle: sends the agent a browse task (read X, write journal, update beliefs)
- Every 6th cycle: sends a tweet task
- Once per day: runs the daily block (article, ponder, daily report, git commit + push)
- Vercel auto-deploys the website on push

**Checkpoint days** (day 3, 6, 9, …) also produce `checkpoints/checkpoint_N.md`.

Use `caffeinate` to prevent macOS from sleeping:

```bash
caffeinate -i bash runner/run.sh
```

## What gets published

Every daily session pushes to GitHub, triggering a Vercel redeploy:

| File | When | What |
|---|---|---|
| `journals/YYYY-MM-DD_HH.html` | Every hour | Raw observations, screenshots, footnotes |
| `daily/belief_report_YYYY-MM-DD.md` | Once per day | Axes created/updated, reflection |
| `articles/YYYY-MM-DD.md` | Once per day (if triggered) | Long-form essay on a topic |
| `ponders/latest.md` | Once per day (if triggered) | Short reflective piece |
| `state/ontology.json` | Every cycle | All belief axes with scores |
| `checkpoints/checkpoint_N.md` | Every 3 days | Full worldview snapshot |

All articles are also permanently archived to **Arweave** via Irys.

## Troubleshooting

**Gateway not starting**
```bash
openclaw gateway status
openclaw gateway start
openclaw doctor
```

**X session expired**
The agent will re-authenticate automatically using `X_USERNAME` / `X_PASSWORD` from `.env`.

**Agent not writing files**
Check `openclaw agents list` — the `x-hunter` agent should be listed.
Re-run `bash runner/setup.sh` if missing.
